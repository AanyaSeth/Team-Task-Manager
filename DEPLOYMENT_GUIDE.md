# Deployment Guide

This guide covers deploying the Node.js backend to Railway and the React frontend to Vercel.

## 1. Backend Deployment (Railway)

1. Create a GitHub repository and push your code (or just the `backend` folder).
2. Go to [Railway.app](https://railway.app/).
3. Click "New Project" -> "Deploy from GitHub repo".
4. Select your repository. If you pushed the entire monorepo (both backend and frontend), you need to configure the root directory.
   - In Railway dashboard, go to Settings -> General -> Root Directory, and set it to `/backend`.
5. Environment Variables:
   - Go to the Variables tab.
   - Add `PORT` (Railway usually handles this, but you can set it to `5000`).
   - Add `MONGO_URI` with your MongoDB Atlas connection string.
   - Add `JWT_SECRET` with a secure random string.
6. Once deployed, Railway will generate a public URL for your API. Copy this URL (e.g., `https://my-backend-production.up.railway.app`).

## 2. Frontend Deployment (Vercel)

1. Go to [Vercel.com](https://vercel.com/) and click "Add New Project".
2. Select the same GitHub repository.
3. Configure the project:
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend`
4. Environment Variables:
   - Add `VITE_API_URL`. Set the value to the API URL generated by Railway (e.g., `https://my-backend-production.up.railway.app/api`).
5. Click "Deploy".
6. Vercel will build and deploy the React app and provide you with a production URL.

## 3. MongoDB Atlas Configuration

Ensure your MongoDB Atlas cluster allows incoming connections from your deployed backend:
1. Go to your MongoDB Atlas dashboard -> Network Access.
2. Ensure you have IP `0.0.0.0/0` (Allow access from anywhere) added, because Railway's outgoing IPs can change.

## Verification
- Navigate to your Vercel URL.
- Register an admin user.
- Try creating a project and a task.
- Ensure that tasks can be updated.
